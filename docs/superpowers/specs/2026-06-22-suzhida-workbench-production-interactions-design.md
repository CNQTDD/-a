# 诉智达工作台生产化交互设计

## 1. 目标

本设计面向诉智达前端工作台的交互生产化改造，解决当前页面仍然停留在 demo 层面的几个核心问题：

- 状态变更按钮缺少统一治理，容易误触发
- 缺少提交中禁用、确认弹窗、恢复重试和错误提示
- 反馈动作仍偏本地演示逻辑，未完整走后端接口
- 同投诉复用、归档、失败恢复等规则没有统一落到前端状态层

本轮只聚焦前端交互壳层和与之配套的最小 API 接入，不扩展新的业务域模型，不改动后端工作流主架构。

## 2. 用户确认的交互原则

- 所有会改变会话状态的按钮都需要二次确认
- `启动处置流程` 发生可恢复错误时自动重试一次
- 其他状态变更动作不自动重试，只允许人工再次确认后重试
- 同一投诉内容再次提交时，优先复用未归档会话

## 3. 设计结论

采用“Pinia 统一动作治理 + 组件纯展示”的方案。

- `store` 负责动作状态、确认上下文、错误提示、重试入口和同投诉复用判断
- 页面视图负责编排，不再分散编写按钮业务规则
- 表单和操作组件只接收可点击性、忙碌态和事件回调
- 新增通用确认弹窗与操作提示条，作为工作台的统一交互外壳

该方案能在不重构现有技术栈的前提下，把高风险动作治理、状态一致性和生产化可维护性收口到同一层。

## 4. 交互模型

### 4.1 动作类型

本轮纳入统一治理的动作：

- 启动处置流程
- 采纳方案
- 编辑后采纳
- 驳回处理
- 归档会话

### 4.2 动作状态

每个动作统一进入以下生命周期：

- `idle`：空闲
- `confirming`：等待用户确认
- `submitting`：请求进行中
- `retryable`：可重试失败
- `failed`：失败，需人工处理

### 4.3 风险规则

- 所有状态变更动作点击后先弹确认框
- 提交中按钮必须禁用，避免重复触发
- 提交成功后，页面给出明确结果提示
- 提交失败后，页面给出动作级错误文案，而不是静默失败

## 5. 会话复用与归档规则

- 同投诉文本匹配时，只复用“未归档会话”
- 命中可复用会话时，直接切回原会话，并提示“已切回未归档会话”
- 已归档会话不参与默认复用
- 归档按钮只对 `resolved` 或 `failed` 会话可用

## 6. 异常恢复规则

### 6.1 启动流程

- 创建会话失败：自动重试一次
- 发送消息失败：自动重试一次
- SSE 连接中断：保留会话并提示“流程已启动，可重新连接事件流”

### 6.2 反馈与归档

- 采纳、编辑后采纳、驳回、归档均不自动重试
- 用户需再次点击并重新确认后才可重试

## 7. 前端边界调整

### 7.1 Store

`frontend/src/stores/complaint.ts`

负责：

- 保存动作状态和待确认动作
- 统一会话复用判断
- 记录操作提示和错误信息
- 管理归档与反馈后的前端状态更新

### 7.2 视图

`frontend/src/views/ComplaintWorkbench.vue`

负责：

- 调度启动流程、反馈和归档动作
- 把动作状态传给各组件
- 承载通用确认弹窗和提示条

### 7.3 组件

- `ComplaintComposer.vue`：显示启动按钮忙碌态
- `FeedbackBar.vue`：显示采纳、编辑、驳回、归档及其禁用态
- `SessionSidebar.vue`：显示归档后的列表变化
- 新增 `ActionConfirmDialog.vue`
- 新增 `ActionNoticeBar.vue`

## 8. API 使用

- 启动流程继续调用 `createComplaintSession` 与 `sendComplaintMessage`
- 人工反馈改为调用 `submitComplaintFeedback`
- 反馈动作需从前端枚举映射到后端 `accepted / edited / rejected`
- 归档先保留为前端归档态，不新增后端接口

## 9. 测试策略

### 9.1 Store 单测

- 确认态进入与清理
- 启动流程自动重试一次
- 反馈失败进入人工重试
- 已完成但未归档会话仍可复用
- 归档后不再参与复用

### 9.2 组件集成测试

- 高风险按钮必须先弹确认框
- 提交中按钮禁用
- 失败提示和重试提示可见
- 驳回原因为空时不能提交

### 9.3 页面流测试

- 连续点击启动不会重复建会话
- 反馈动作会走真实 API mock
- 同投诉会话复用命中后不会新建会话

## 10. 本轮非目标

- 不新增结束会话后端接口
- 不引入权限系统和登录态
- 不改造整套历史页面文案体系
- 不在本轮加入完整审计日志面板
