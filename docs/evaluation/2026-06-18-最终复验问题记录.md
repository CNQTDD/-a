## 补充问题 10：同一投诉重复点击会反复创建新的会话 ID

- 发现阶段：前端手工联调投诉处置工作台
- 现象：
  - 对同一条投诉内容连续点击“启动处置流程”后，左侧会话列表不断出现新的 UUID
  - 用户感知上像是“同一个问题被系统拆成了多个独立会话”
- 影响范围：`Task 17`、`Task 19`、`Task 20`
- 根因：
  - 前端 `ComplaintWorkbench.vue` 每次提交都会先调用创建会话接口，没有复用本地已存在的未完成会话
  - 前端创建会话时没有透传 `Idempotency-Key`，导致后端幂等创建逻辑无法生效

- 解决方案：
  - 为同一份投诉草稿生成稳定的 `Idempotency-Key`，创建会话时显式传给后端
  - 在前端 store 中增加“同投诉文本未完成会话复用”能力，命中后直接切回原会话，不再重复建会话
  - 扩展前端单元测试，覆盖“透传幂等键”和“重复点击不重复建会话”两个回归场景
- 相关测试：

  - 页面只在动作真正进入 `submitting` 时禁用确认弹窗按钮
  - 保留确认态用于展示和流程控制，不再用于阻塞用户点击“确认”
